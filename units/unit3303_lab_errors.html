Lab - Errors

  • 45 minutes
  • Paired Exercise
  • Lab Objectives
    • Create errors
    • Find errors
    • Use try-catch blocks

Error Lab

To start this lab, find a partner. You will complete this first half of this lab separately, and then the lab will provide directions on how to work with your partner!

1. Create a new file in your ROOT\test folder called errors.jsp

2. In this file, create a small program with the following instructions:

Your program is going to perform wedding reception calculations.  It should figure out how many tables are needed, and then figure out the total cost of the reception assuming it only includes the cost of tables, food, and decorations.

Here are some exercise guidelines:

Your program should provide variables for the following items:
* The number of guests wedding partner A invites
* The number of guests wedding partner B invites
* Seats per table
* Cost per table
* Cost per reception dinner plate
* Total cost of decorations

Your program should calculate and print the following items:
* Total number of tables needed
* Total cost for tables
* Total cost for food
* Total cost of decorations
* Total wedding reception cost

Create Errors

Hopefully, you have written a beautifully functional wedding reception calculator. Now, we’re going to mess it up and hand it over to your partner to fix.

3. Syntax errors. Create five syntax errors in your code. They can be of any flavor you choose, but they must prevent compilation.

Some dastardly ideas for you:
* Misspelled variables
* Missing semicolons
* Incorrect or missing keywords (such as int or double)

4. Logical errors. Create three logical errors in your code. They can be of any flavor you choose, but they shouldn’t prevent compilation.

Some evil genius ideas for you:
* Incorrect math
* Using the wrong variables in math formulas

5. Run-time errors. Create one run-time error in your code.

Trade Programs

6. Exchange programs with your partner!

Fixing Errors

7. Try running the program. If your partner has completed their task, it should fail.

8. Scan the program and read the code from the first line to the end. See if you can follow the code, spotting and fixing any syntax errors as you go. Keep running the program until all compilation errors are fixed.

Look at the type of error the page is reporting. Is it a compilation error or processing error? If you find a processing error, that means that compilation has succeeded. Use try-catch blocks to contain any processing errors.

9. Look at the output of the program and see if you can spot any math that seems off. How does the program line up with your calculations on paper?

10. If the program’s output is incorrect, rescan the code and see if you can spot any logical errors. Try different numbers of guests or seats per table, for example, to help spot more logical errors.

Congratulations, we’re finished with this lab. See you in the debrief!
