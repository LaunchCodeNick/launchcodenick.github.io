Lecture - HTML Forms

  • 30 minutes
  • Learning Objectives
    • Learn the concept of HTML forms
    • Learn a textbox input control
    • Learn querystring parameters
    • Learn URL encoding
    • Learn external “get” form submission

User Input

So far, you’ve learned how to display text, images, and links on a webpage. Many of these elements are referred to as output, which are things that our code will display to a user.

But what if you want to collect information from a user? For example, Google.com has a text box to ask what a user wants to search. Elements that a user can type or change the state of are often referred to as input.

Textbox

The textbox might be the most common input element across all of programming. Here in HTML, we’re going to use the following code to put one on our page.

<input type="textbox" name="a name for your textbox" />

Notice that there is no separate closing tag! But the opening tag closes itself with a />

1. Create a new html file called textbox.html and write the basic page structure elements and place a textbox named “search” in the body.

    After you've tried, if you need a hint, you can click on the example tab.

<!doctype html>
<html>
<head></head>
<body>
    <input type="textbox" name="search" />
</body>
</html>

2. View your page in a web browser. Try typing the word “iPhone” into the textbox and try pressing enter.

You should be able to type text in the box, but the enter key should not do anything.

Forms

In order to bring your HTML input elements to life, you’ll need to put them inside a form. Once a form wraps the input elements on a page, the enter key will submit the webpage which either triggers a page refresh or takes us to a completely new webpage.

We’re going to use the following code to put one on our page:

<form method="get">
    <input type="textbox" name="search" />
    <input type="submit" value="Go!" />  <!-- good practice to put a "submit" button in each form -->
</form>

3. Update your code, type the word “iPhone” into the textbox and press enter again.

4. Look at the address of your webpage in the web browser. What do you notice is different?

When a form’s action is set to “get”, the values of all input elements in a form get placed on the web address. We call these querystring parameters.

5. Place a second textbox on your form named “search2”.

6. Type the word “iPhone” in the first textbox, and type the word “radio” in the second textbox. Press the enter key and look at your browser’s address bar.

When a form’s method is set to “get”, a “?” is added to the end of the original web address to signal the beginning of querystring parameters. Then each input element gets added with a name=value convention, and multiple input elements are separated with a “&”.

For example:

http://samplewebsite.com/searchpage.html?search1=iPhone&search2=radio

Special Characters

There are some text characters that don’t work as part of a web address. This is often because they are characters used in internet addresses. So, if your querystring parameter value contains one of these special characters, they will get translated to a special code, a process we call url encoding.

To see an example, try the following search terms and review your web browser bar to see how the querystring parameter looks.

7. Type “test space” into the first textbox and “test:colon” into the second textbox

You should see the space character encoded as a + sign and the : character encoded as %3A

8. Type “test&ampersand” into the first textbox and “test@at” into the second textbox

You should see the & character encoded as %26 and the @ character encoded as %40

9. Type “test/slash” into the first textbox and “test\backslash” into the second textbox

You should see the / character encoded as %2F and the \ character encoded as %5C

10. Type “test+plus” into the first textbox and “test%percentage” into the second textbox

You should see the + character encoded as %2B and the % character encoded as %25

The presence of a % in an address bar usually signals that a parameter has been url encoded.

External Websites

Did you know that many Internet websites use “get” forms and querystring parameters?

11. Head to http://www.amazon.com and enter iPhone as a search term. Press enter and look at the address bar. Can you find the name of the input element and see your search term in the address bar?

At the time of writing this lesson, the website url looks like this:

https://www.amazon.com/s/ref=nb_sb_noss?url=search-alias%3Daps&field-keywords=iphone

You can see very easily that the name of the textbox was “field-keywords” and that my search term is attached to it. There are also several other querystring parameters that Amazon is using, but if you try searching a few different things, you will see these values aren’t changing, meaning they aren’t affected by a user’s input.

Submitting a Form to External Websites

Did you know that we can make our textbox.html page interact with Amazon.com?

12. Update your form tag with an action attribute with the enter Amazon address before the “?”, and change the name of the first textbox to “field-keywords”:

<form method="get" action="https://www.amazon.com/s/ref=nb_sb_noss">
    <input type="textbox" name="field-keywords" />
    <input type="textbox" name="search2" />
    <input type="submit" value="Go!" /> 
</form>

13. Type “iPhone” in the first textbox and “radio” in the second textbox. Press enter or click the form button and see what happens.

Your search page should work with Amazon! The second textbox has a name that Amazon doesn’t recognize, so its value is ignored by their program. However, the first search box should control the search results from Amazon.

Challenge Question

In your own words, what is happening and how did we make this happen?

We visited Amazon.com and tried using their search engine.  We noticed they use querystring parameters, because we could see the search terms go up into the address bar.  We created our own web form and gave a textbox the exact same name that we saw in the Amazon address bar ("field-keywords").  Then we told our form, when submitted, to pass all of its infomation via querystring to an Amazon web address.
