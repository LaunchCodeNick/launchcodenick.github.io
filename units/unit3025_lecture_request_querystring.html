Lecture - Request Querystring

  • 15 minutes
  • Lecture Objectives
    • Learn to retrieve a form parameter

Querystring Parameter

If you recall, when a HTML form is submitted via ‘get’ method, it puts form data on the querystring like so:

http://localhost:8080/test/request.jsp?color=blue

Now that you now a little bit of JSP and Java, you can use a command to retrieve a querystring parameter in your Java code.

1. Create a file in your ROOT/test folder called request.jsp and use the following starter code:

<!doctype html>
<html>
<head><title>Requesting Parameters</title></head>
<body>
    <form method="get">
        <input type="textbox" name="color" />
        <input type="submit" value="Submit" />
        <%
            String color = request.getParameter("color");
            out.println("the color is: " + color);
        %>
    </form>
</body>
</html>

2. Visit your webpage at: http://localhost:8080/test/request.jsp

What does the page say the color is?

When your querystring parameters are missing or empty, they are recorded as a null value in Java. Not the word ‘null’, but an actual value of null.

3. Try typing different names of colors into the textbox and see what prints out.

You should see your values in both the querystring and printed out on the page.

4. Try changing the value in your browser bar to see if it will work that way too. Visit: http://localhost:8080/test/request.jsp?color=robinseggblue

5. Try this version that might work a little easier on the eyes: http://localhost:8080/test/request.jsp?color=robins+egg+blue

Strings, Strings, Strings

It’s important to know that everything on the querystring is going to be retrieved as a String, even numbers!

6. Change your code to the following:

<!doctype html>
<html>
<head><title>Requesting Parameters</title></head>
<body>
    <form method="get">
        <input type="textbox" name="number" />
        <input type="submit" value="Submit" />
        <%
            int myNumber = request.getParameter("number");            
        %>
    </form>
</body>
</html>

You should get a variable type mismatch error! request.getParameter only returns Strings! Strings for everything!

7. Let’s capture a number as a String then convert it to an integer using Integer.parseInt();

<!doctype html>
<html>
<head><title>Requesting Parameters</title></head>
<body>
    <form method="get">
        <input type="textbox" name="number" />
        <input type="submit" value="Submit" />
        <%
            String myNumberAsString = request.getParameter("number");            
            
            if (myNumberAsString != null) { 
                int myNumber = Integer.parseInt(myNumberAsString);
                out.println("the number is: " + myNumber);
            }
        %>
    </form>
</body>
</html>

8. Try entering 2 into the textbox and submitting the form.

9. Try entering X into the textbox and submitting the form.

We need to be careful when processing a String that we expect will contain a number. If bad data slips into the String, it could cause a major error. We will learn how to handle this in a future lab.
