Algorithms

Sometimes, in programming, we are asked to solve little puzzles that involve mathematical computations.

The step-by-step solution we use code to articulate is called an algorithm.

Many technical interviews for computer programming jobs will test you ability to solve algorithms because they are a true representation of the challenge that faces many programmers on a daily basis.

In this course, we will try to give you plenty of practice with algorithms. You should expect the algorithms to get more difficult as we go.

Unit Testing and Error Handling

What did you learn from testing more cases at the end of the lab?

Whenever we work with sections of code, particularly algorithms, it will be a good practice to think through examples of good and bad user inputs. As we move through this course, you will learn to anticipate bad inputs and write code that gracefully handles them. Anticipating errors and handling them is referred to as error handling (obviously!).
