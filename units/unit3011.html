Static vs. Dynamic Web Pages

So far, you’ve built “static” web pages by writing HTML to files. Assuming you are finished with these files, you will always see the same thing when you open them - this is the definition of “static.”

But the most interesting pages on the Internet are not static, they are “dynamic.” Take an Amazon search results page, for example. If you search for iPhone cases, you will see a web page with iPhone case information. If you search for radios, you will see a web page with radio information. Although the web page itself is the same file, it will display differently depending on certain factors, such as user input. We call this a dynamic web page.

The Architecture of Static and Dynamic Web Pages

Static web pages can be very easy to build. In fact, you’ve been building them with a text editor and then opening them directly with a web browser! Static web pages are built directly with HTML, CSS, and JavaScript (you have not learned the last two!)

A dynamic web page, however, often requires something called a back-end programming language. This back-end language can create, edit, manipulate, or remove HTML, CSS, and JavaScript to change what a web page displays, as the page is actually being requested!

The following can be used as back-end programming languages:

  • C#.NET
  • Java
  • Node.JS
  • Perl
  • PHP
  • Python
  • Ruby

Obviously, we will use Java for this course!

A High-Level Example

A dynamic web page might include HTML and a bit of back-end Java together in the same file. Notice that the Java code is written between <% and %>.

<!doctype html>
<html>
<head></head>
<body>
    <% // this is a made up Java function, but go with it!
       makeTenTextboxes();  %>
</body>
</html>

When this page is called by a user, an application called a “web server” will execute the code on this page and return a final HTML file to the user:

<!doctype html>
<html>
<head></head>
<body>
    <input type="textbox" name="textbox1" />
    <input type="textbox" name="textbox2" />
    <input type="textbox" name="textbox3" />
    <input type="textbox" name="textbox4" />
    <input type="textbox" name="textbox5" />
    <input type="textbox" name="textbox6" />
    <input type="textbox" name="textbox7" />
    <input type="textbox" name="textbox8" />
    <input type="textbox" name="textbox9" />
    <input type="textbox" name="textbox10" />
</body>
</html>

It’s this type of technology that lets the Amazon search results page display iPhone items if you searched for iPhones, or radio items if you searched for radios. You can use Java code to dynamically write HTML code when the page is requested!

Web Servers

Going forward, we will need a web server to make dynamic web applications. The web server is going to change how we open/locate our web pages. We won’t write a simple text file any longer and then open it directly in a web browser.

Instead, we will put our files inside the directory of a web server. And we will view them by making a request through the web server’s address.

When we set up a web server on your computer, we call that a local installation. The name of your web server will almost certainly be “localhost” and you will view pages on it by visiting “http://localhost:8080/examplepagename.html”

You will install a web server in your next lab and start to understand how this is all set up!

Apache Tomcat

You may be wondering: what’s a good web server to use for Java web applications? How would I know this? What if I choose to make applications in another language someday, what server would be good then?

Typically, you’ll want to stick with the crowd on this sort of thing.

1. Use Google to search for the the most common web servers for Java applications.

You should find that Apache Tomcat is the most common web server for Java applications, so that is what we will use in this course.

Java

In order for Apache to work with Java, we also need to install Java!

There are two types of Java installations you will find: the Java Run-time Environment (JRE) and the Java Development Kit (JDK).

Computers that only need to run Java applications can install the JRE. But computers that need the ability to run and develop Java applications should install the JDK (which includes the JRE). So we will install the Java JDK in this course.
