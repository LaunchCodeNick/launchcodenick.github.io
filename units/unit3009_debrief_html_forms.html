Layout Choices

When building a web page, you may need to make choices about how things are laid out.

This shopping cart has Hamburger first, followed by Fries, followed by Soda, followed by Ice Cream.

Is this a better sequence than Soda, Ice Cream, Fries, Hamburger?

Why is one sequence better than other?

Activity

In this last lab, you only worked with one group of radio buttons. What did you notice about the names of the buttons?

How would you create two different groups of radio buttons? For example, how would you create a form with a group of radio buttons for car colors, and a group of radio buttons for manual or automatic transmission?

You can click the Answer tab to see how to create two groups of radio buttons on the same web page.

<input type="radio" name="car_color" value="black" checked /> Black
<input type="radio" name="car_color" value="blue" /> Blue
<input type="radio" name="car_color" value="red" /> Red
<input type="radio" name="car_color" value="white" /> White

<input type="radio" name="car_transmission" value="automatic" checked /> Automatic
<input type="radio" name="car_transmission" value="manual" /> Manual

User Experience

In this last lab, we used readonly textboxes to display the price of each item.

When a user sees a textbox, what will they expect?

Users expect that they can edit the information in a textbox. A readonly box canâ€™t be edited or even clicked into, and this experience can be confusing or awkward for a new user. Some potential solutions include using a different type of input control, or making the textbox look less like a textbox by removing its border.
