One Lab Solution

<!doctype html>
<html>
<head><title>Wedding Reception Calculator</title></head>
<body>

<%    
    int invitesA = 40;
    int invitesB = 25;
    int invitesTotal = invitesA + invitesB;
    
    double costPerTable = 10.00;
    double costPerPlate = 20.00;
    double costDecorationsTotal = 1000.00;
    double costTablesTotal = 0;
    double costFoodTotal = 0;
    double costGrandTotal = 0;
    
    int seatsPerTable = 8;
    int tablesTotal;
    
    // calculate number of tables
    tablesTotal = invitesTotal / seatsPerTable; // integer math drops decimal
    
    try {
        // we're going to round tables up 1 if there was a decimal for a partial tables
        // one approach: use math test
        if (tablesTotal * seatsPerTable != invitesTotal) {
            tablesTotal = tablesTotal + 1;
        }
        
        // a second approach: use the mod (%) operator
        if (invitesTotal % seatsPerTable != 0){ // if there's a remainder, round up
            tablesTotal = (invitesTotal / seatsPerTable) + 1;
        } else {
            tablesTotal = (invitesTotal / seatsPerTable);
        }
    } catch (Exception e) {
       out.println("There is an error calculating the number of tables.");        
    }
    
    try {
        // calculate all my costs
        costTablesTotal = costPerTable * tablesTotal;
        costFoodTotal = costPerPlate * invitesTotal;
        costGrandTotal = costTablesTotal + costFoodTotal + costDecorationsTotal;
    } catch (Exception e) {
        out.println("There is an error calculating costs.");
    }
    
    try {
        // display results
        out.println("<h1>Wedding Reception Calculator</h1>");    
        out.println("Number of Wedding Reception Guests: " + invitesTotal + "<br>");
        out.println("----------------------------------------------------" + "<br>");
        out.println("Total Table Costs: $" + costTablesTotal + "<br>");
        out.println("Total Food Costs: $" + costFoodTotal + "<br>");
        out.println("Total Decoration Costs: $" + costDecorationsTotal + "<br>");
        out.println("----------------------------------------------------" + "<br>");
        out.println("Grand Total: $" + costGrandTotal);
    } catch (Exception e) {
        out.println("There is an error displaying results.");        
    }
%>
</body>
</html>

Multiple Try-Catch Blocks

Notice that the lab solution file uses multiple try-catch blocks that are grouped around logical sections: calculating tables, calculating costs, and displaying results.

It’s a good practice to use try-catch blocks around smaller sections of code that belong together. Later in the course, you’ll also be exposed to anticipating specific kinds of errors. Right now, we are catching Exception which is the most general catch-all kind of exception in Java, but there are many more specific kinds.

Challenge Question

If the first try-catch block errored, do you think it’s wise that the program continues on to calculate costs and display results? How might you change the solution?

Click the 'Opinion' tab when you're ready to hear one perspective.

There's no exactly right answer here, but it doesn't seem to make sense that the rest of the program should continue if a previous block errors.  There are some contrived ways to manage this with if statements and variables, but the cleaner way to approach is this is with multiple 'methods.'  We will learn about methods in the next lecture.
