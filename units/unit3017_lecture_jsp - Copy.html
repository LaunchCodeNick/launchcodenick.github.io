Java Servlet Page (JSP)

A JSP is a HTML file that contains blocks of Java code within it. The Java code is executed when the page is requested and can change the final HTML output that is delivered to a web browser. This is what makes JSP a dynamic web page.

Hello World!

In the lab to test your Apache Tomcat installation, you learned that the Apache ROOT folder is where your web files will need to go.

You also created your first Hello World JSP file and named it index.jsp and placed it a ROOT\test folder. Let’s make one more!

1. Start up Apache Tomcat.

Open a command line or terminal window and start your Apache Tomcat server.

On Windows, you should be able to do this by typing ‘startup.bat’

On Mac, you should be able to do this by typing ‘sh startup.sh’

2. Create a new file in your Apache ROOT\test folder named ‘hello_world.jsp’.

3. Use the following code for your hello_world.jsp file.

<!doctype html>
<html>
<head><title>Hello World JSP!</title></head>
<body>
    <%= "<b>Hello</b> World!" %>
    <% out.println("<b>Hello</b> World 2!"); %>
</body>
</html>

4. Remember that your Apache is hosting web files at http://localhost:8080 . What should the web address of this Hello World file be?

Click the 'Answer' tab to see the correct web address.

It should be http://localhost:8080/test/hello_world.jsp

5. Visit your Hello World page and examine the output.

Notice that the webpage was able to process the HTML tag with no trouble.

6. Right click on the web page and choose ‘View Page Source’. You can use this feature to look at the final HTML that was delivered to the web browser.

Both lines of Java code printed similar Hello World text. The first line used a <%= opening tag. Notice the equal sign! When you use an equal sign, you are saying that you want some text inserted into the HTML page.

When you use the <% without the equal sign, you can use multiple lines of Java to do something, which will eventually allow you to write much more complex code.

7. Let’s quickly change your code to the following:

<!doctype html>
<html>
<head><title>Hello World JSP!</title></head>
<body>
    <%= "<b>Hello</b> World!" %>
    <% 
        out.println("<b>Hello</b> World 2!"); 
        out.println("<b>Hello</b> World 3!"); 
        out.println("<b>Hello</b> World 4!"); 
        out.println("<b>Hello</b> World 5!"); 
    %>
</body>
</html>

Notice that we could put multiple lines of Java inside one group of <% %> tags.

Strings

In the world of programming, any kind of text value is generally referred to as a string. You can think of this as a string of individual text characters.

Whenever we use string text in Java, it needs to be wrapped in double quotation marks (this one: “).

8. Remove the quotations from the following line in your JSP page and see what happens when you refresh the page.

<!doctype html>
<html>
<head><title>Hello World JSP!</title></head>
<body>
    <%= <b>Hello</b> World! %> <!-- no more double quotations here! -->
</body>
</html>

You should get a compile error. The web server is unable to process the Java in this web page.

9. Multiple individual strings can be added together by using a + operator. Try the following code:

<!doctype html>
<html>
<head><title>Hello World JSP!</title></head>
<body>
    <%= "<b>Hello</b>" + " " + "World!" %> <!-- add strings together with a + operator -->
    <% out.println("<b>Hello</b>" + " " + "World 2!"); %>
</body>
</html>

Semicolons

So what were those semicolons about?

HTML is largely a language that says “this is how a web page should look.” By contrast, Java is a language we’ll use to run multiple steps in a particular sequence.

If you like analogies, we can think of baking a cake. HTML is sort of like the beautiful picture of the finished cake. Java is like the individual steps required to make the cake.

When you see a recipe, it usually comes in specific steps.

1. Preheat the oven
2. Grab a bowl
3. Crack some eggs into the bowl.

In Java, each step that needs to run is separated by a semicolon:

Preheat the oven;
Grab a bowl;
Crack some eggs into the bowl;

Variables

Remember Algebra class? How we would write things like x = 7. Well in Java (and many other languages), we can do a similar thing.

10. Let’s make a String variable called myfirstvariable and set it equal to “Hello World!”.

<!doctype html>
<html>
<head><title>Hello World JSP!</title></head>
<body>
    <%
        String myfirstvariable = "Hello World!";
        out.println(myfirstvariable); // don't use double quotes around a variable name!
        out.println("myfirstvariable"); // see what happens if we do...
    %>
</body>
</html>

11. Let’s make two String variables and add them together:

<!doctype html>
<html>
<head><title>Hello World JSP!</title></head>
<body>
    <%
        String variable1 = "Hello";
        String variable2 = "World!";
        out.println(variable1 + " " + variable2);
    %>
</body>
</html>

Numbers

In this lecture, we’ve talked about text called Strings. But that is just one “data type” in Java. We’ll often want to work with numbers too.

There are a few different kinds of numeric data types. These types often have nuanced characteristics. Some can’t hold decimals well (1.78 for example), some can’t hold gigantic numbers (100,000,000,000,000 for example).

Why are there different kinds at all? These flavors of numeric data types often exist to manage how much computer memory a program will consume. It takes more computer memory to store gigantic numbers than small ones, and more memory to store numbers with decimals than non-decimal numbers. We won’t run into memory problems in this course, but this is a good thing to know!

13. Let’s write some code that uses numeric variables:

<!doctype html>
<html>
<head><title>Hello World JSP!</title></head>
<body>
    <%
        int number1 = 10; // no quote marks on numeric values!
        int number2 = 1;
        int total = number1 + number2;
        
        out.println(number1 + number2);
        out.println("<br>");
        out.println("the total is: " + number1 + number2);
        out.println("<br>");
        out.println("the total is: " + (number1 + number2));
        out.println("<br>");
        out.println("the total is: " + total);
    %>
</body>
</html>

Challenge Question

Of the four methods that the previous code example used, the second one did not work.

Why do you think that is?

Click the 'Answer' tab to discover the issue.

When adding different data types together, in this case a String of "the total is: ", and numeric values of 10 and 1, Java wants to convert each variable that follows to a String.

Java turned the 2nd example into "the total is: " + "10" + "1"

The reason the 3rd example works is because Java processes (number1 + number2) first, and then it converts it to a String.  The parentheses helped Java to understand what to do first.
