Evaluating Numeric Expressions in Java

In order for us to use Java code that will run in just some cases, we will need to learn a combination of If statements and evaluating expressions.

For example, if the temperature is less than 60 degrees, I’ll wear a sweater. If the temperature is more than 80 degrees, I’ll wear shorts.

1. Lets create a file in our ROOT\test folder called if.jsp

2. Create the barebones HTML code for it, and then view the webpage in your browser.

3. Use the following code in your page and see what prints out:

<%
    out.println(80 > 60);    
    out.println(80 <= 60);
        
    // in Java, we test for numerical equality using two equal signs
    out.println(80 == 60); 
%>

You should see the results of our three expressions: true, false, and false. Yes, 80 is greater than 60, it’s not less than or equal to 60, and it’s not equal to 60.

4. This also works with variables. Let’s try the following code:

<%
    int x = 80;
    double y = 60.0;
    
    out.println(x > y);    
    out.println(x <= 10);
        
    // in Java, we test for numerical equality using two equal signs
    out.println(x == y); 
%>

Evaluating Strings in Java

When we want to evaluate whether two Strings are equal, however, we need to use slightly different code. The underlying reason is because numeric data types are primitive data types in Java, and Strings are not. Strings are actually a form of something called an Object, which is a very important concept you will learn later in this course.

For now, you just need to know that you can evaluate Strings using the method below:

<%
    String animal = "lion";
    String kingOfTheJungle = "lion";    
     
    out.println(animal.equals(kingOfTheJungle));        
    out.println(kingOfTheJungle.equals(animal));        
%>

If Statements in Java

In programming, we often need to execute code in a certain condition.

We use an ‘if’ statement and then a set of curly braces { }. Any statements inside the { and }, immediately after an If statement will execute if the statement evaluates as true.

5. Try the following code:

<%
String animal = "lion";

if (animal.equals("lion") == true) {
    out.println("ROAR!");
}
%>

You should see a mighty roar!

6. What if the animal isn’t a lion? Try the following code:

<%
String animal = "mouse";

if (animal.equals("lion") == true) {
    out.println("ROAR!");
} else if (animal.equals("mouse") == true) {
    out.println("eek!");
}
%>

We can test for more conditions by using an ‘else if’ statement.

7. What if the animal is something we probably couldn’t predict? Try this:

<%
String animal = "a ridiculously tall Giraffe named Sam";

if (animal.equals("lion") == true) {
    out.println("ROAR!");
} else if (animal.equals("mouse") == true) {
    out.println("eek!");
} else {
    out.println("whoa!");
}
%>

If you need a catch-all clause in case you have no idea what else might come, or it really is a situation of lion or anything else at all, you can use an ‘else’ statement at the end of your if-block.

8. If statements can also test for multiple conditions using AND or OR type operators. In Java, we use && to signal an AND operator, and we use || to signal an OR operator.

<%
boolean isBig = true;
boolean isScary = true;
String animal = "";

if (isBig && isScary) {
    animal = "lion";
} else if (isBig == true && isScary == false) {
    animal = "panda bear";
} else if (isScary == true) {
    animal = "snake";
} else {
    animal = "butterfly";
}

if (animal.equals("lion") || animal.equals("snake")) {
    out.println("Run away from a " + animal + "!");
} else {
    out.println("I love this " + animal + "!");
}
%>

Variable Scope

The last concept in this lecture is on a topic called Scope. Scope is used to define the concept that variables don’t exist everywhere all the time. They only exist within a certain scope of code.

9. Try this example to see what happens:

<%
boolean isTrue = true;

if (isTrue) {
    String scopeExample = "watch this";
}

out.println(scopeExample);
%>

You should get an error that says “scopeExample cannot be resolved to a variable.” Since scopeExample was declared inside a set of { } braces, and so it isn’t accessible outside of them.

10. Now, try this example to see what happens:

<%
boolean isTrue = true;

if (isTrue) {
    String scopeExample = "watch this";
    if (isTrue) {
        out.println(scopeExample);
    }
}
%>

This example works because scopeExample was declared at a higher level than it’s being used. It was already declared before a statement inside of a nested { } referenced it.

Challenge Question

Which of the following variables are being referenced out of scope?

<%
boolean isTrue = true;
String apple = "apple";

if (isTrue) {
    String cherry = "cherry";
    if (isTrue) {
        String grape = "grape";
        out.println(cherry);
    }
    out.println(grape);
}

out.println(apple);
%>
 
Click the 'Answer' tab after you've tried to complete this challenge!

Grape is the only variable that is referenced out of scope.  It was declared inside the most nested set of { }, and then referenced outside of them. 

Both apple and cherry variables are declared outside of { } tags that they were referenced inside of - which is totally fine!
