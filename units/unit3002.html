Lecture - Introduction to HTML

  • 30 minutes
  • Learning Objectives
    • Learn the concept of HTML tags and attributes
    • Learn that a web browser renders the HTML
    • Learn how to structure an HTML page
    • Learn that indentation makes code easier to read
    • Learn that non-displayed comments can store programmer notes

Webpage

A webpage is a document that is commonly written using HTML (hypertext markup language) that is viewed using a web browser such as Google Chrome, Safari, Mozilla Firefox, or Internet Explorer.

Markup Language

A markup language uses tags that annotate text to change behavior or describe content . Two markup languages you’ll encounter frequently are HTML, which is used to write web pages, and XML, which is often used to store program data or configuration settings.

Tags

HTML tags usually follow a standard sequence of opening tag, followed by the text to be tagged, followed by a closing tag.

    <tag>text to be tagged</tag>

Use any text editor to complete the following tasks. We recommend Notepad++ which you can download from the tools page:

1. Create a new text file and write the following text inside the document

    Hello World!

2. Save the file as “helloworld.html”
3. Open the file using a web browser to view it

You should see a very simple webpage with the Hello World! text in it.

4. Add a bold tag to the text in your file with the following code:

    <b>Hello World!</b>

5. Refresh your web browser to see the changes in your helloworld.html webpage.

Your Hello World text should now be in a bold font.

Attributes

Many tags can have one or more optional attributes that go inside the opening tag.

    <tag attribute="value1" attribute2="value2">text to be tagged</tag>

6. Let’s change the font face and size of your text with the following code:

    <font face="Arial" size="20">Hello World!</font>

Your Hello World text should now be in the Arial font and much bigger!

Notice how your web browser did not write any of the new words (font, face, size) to the screen. The browser is built to process HTML tags and render webpages accordingly.

Nested Tags

Markup languages are also capable of nesting tags to apply multiple behaviors to content.

    <tag1><tag2 attribute="value">text to be tagged</tag2></tag1>

7. Let’s nest our two examples with the following code:

    <font face="Arial" size="20"><b>Hello</b> World!</font>

Your Hello World text should now be in the Arial font and much bigger, and just the word Hello should be in bold!

8. Finally, let’s try one extra special tag:

    <sparkle>Make this text shine!</sparkle>

Does your text sparkle? Probably not :(

9. Hmmm, let’s try this variation that uses an attribute instead:

    <font sparkle="on">Make this text shine!</font>

How about now, does your text sparkle? No, it doesn’t?

Challenge Question

How does your web browser know which keywords are legitimate tags and attributes?  
Why did <b> and <font face="Arial" size="20"> work, but <sparkle> and <font sparkle="on"> did not?

Founded in 1994, the World Wide Web Consortium (W3C) is the main international standards organization for the Web.  It was created to develop protocols and guidelines that ensure long-term growth for the Web.

Programmers that develop web browsers (for example, Google developers make Chrome and Apple developers make Safari) do their best to follow standards set by W3C, the World Wide Web Consortium.  This means recognizing a standard list of tags and attributes and rendering them in an expected fashion.  <b>, <font>, and font attributes of "face" and "size" are all recognized standards.  However, <sparkle> and a font attribute of "sparkle" are not!

It's import to know that not every browser follows these standards perfectly.  Some browsers have been programmed to recognize extra tags or attributes, or they can interpret and render tags differently.  

Have you ever encountered a webpage that wasn't working, and someone recommended that you try using a different browser?

Page Structure

One standard set forth by W3C is that HTML pages should be structured with 4 main parts:

  • a line containing HTML version information
  • a <html> tag
  • a header section
  • a body, which contains the document’s actual content

Let’s see how that looks in HTML:

10. Create a new text file called pagestructure.html, type this code, and view it:

    <!doctype html>
    <html>
        <head>
            <title>Look at the browser bar!</title>
        </head>
        <body>
            <font face="Arial" size="20">Hello World!</font>
        </body>
    </html>

You should see a familiar Hello World! on the screen, but look at the title bar of your web browser or browser tab. It should say “Look at the browser bar!” now.

11. Change your code to match what’s below. What do you expect to be different?

    <!doctype html>
    <html>
        <head><title>Look at the browser bar!</title></head>
        <body><font face="Arial" size="20">Hello World!</font></body>
    </html>

Challenge Question

What, if anything, was different between the code examples in the two previous examples?

The answer is that nothing should be different between the two examples!

With HTML, extra spacing, indentation tabs, and new lines do not change how a web page is rendered.

However, these formatting items are used by programmers to make code more readable.  As your web pages grow in size and complexity, it will be easier to maintain them if you use some level of formatting.

Commenting

You can also include comments for yourself or other programmers who may work on your code in the future. These comments will not be displayed by the web browser, but are visible in the source code file only.

Different programming languages have various ways of allowing you to write comments, the following is how it’s done in HTML:

    <!-- this is text that is only seen when looking at the code -->

12. Try writing your own comments for your file, something similar to:

    <!doctype html>
    <html>
        <head>        
            <!-- this is what will display in my browser's title bar -->
            <title>Look at the browser bar!</title>
        </head>
        <body>
            <!-- I wanted to make Hello World! look a lot bigger -->
            <font face="Arial" size="20">Hello World!</font>
        </body>
    </html>
