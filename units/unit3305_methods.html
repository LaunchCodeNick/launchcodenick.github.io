Java Methods

In programming, a method is a block of code that logically belongs together. We often use methods because we find ourselves using the same kind of code repeatedly. Rather than write copies of code all over our program, we can write 1 flexible version of those copies, and then call, or use, that single block of code many times.

Many Java methods perform some steps and then return a value. Look at the following skeleton of a Java method:

returnType methodName(inputParameter1Type inputParameter1Name, inputParameter2Type inputParameter2Name, ...) {
    // some lines of code
    // some more lines of code
    return returnType;
}

Look at this working implementation of this method skeleton:

double calculateTotalCost(int numberOfItems, double costPerItem, double salesTaxPercent) {
    double totalCost = numberOfItems * costPerItem * (1+salesTax);
    return totalCost;  // the variable being returned must match the returnType of the method
}

We are saying we have a block of code called ‘calculateTotalCost’ that requires 3 input parameters to do its job. It needs an integer that represents some number of items, a double that represents the cost per item, and a double that represents the sales tax percent.

If you give this method those 3 items, it will give you something back. It will give you a double (the first word in the method declaration here) that represents the total cost calculated.

Calling a Method

And here is how we would call, or use, that method in our program:

double salesTax = 0.07;

int numberOfTables = 4;
int costPerTable = 20.00;

int numberOfGuests = 100;
int costPerDinner = 25.00;

// use the calculateTotalCost method - example #1
double totalTableCost = calculateTotalCost(numberOfTables, costPerTable, salesTax); // use #1

// use the calculateTotalCost method - example #2
// don't always have to use variables, can pass direct values as well
double totalDinnerCost = calculateTotalCost(numberOfGuests, costPerDinner, 0.07); 

// output examples
out.println(totalTableCost); 
out.println(calculateTotalCost(numberOfTables, costPerTable, salesTax));

In this example, we are calling the method twice. Once to calculate a total cost for tables, and once to calculate the total cost for dinner. You may be wondering, why are we using a method? The calculation seems pretty simple. Eventually, your calculations will become so complex that anything but a method will create extremely long code that is difficult to maintain.

Notice that the names of the variables we pass to the method don’t have to match the names that the method uses (and often will not). The method has a variable named numberOfItems, and we are giving it numberOfTables there in use #1, and numberOfGuests in use #2. You can think of this as the code setting numberOfItems = numberOfTables. It’s not using the variable numberOfTables itself, but rather copying its value into another variable named numberOfItems.

As a last point with this example, since the value of these parameters are being copied into new variables (something we call passing parameters by value), if we make any changes inside the method to the variable, it will not affect any variable values outside of the method. See the example below.

int addTen(int x) {
    x = x + 10; // the x inside the method will be changed
    return (x);
}

int y = 96;
out.println(addTen(y)); // will print 106
out.println(y); // will print 96, y was never altered

int x = 96; // even if the variable is the same name
out.println(addTen(x)); // will print 106
out.println(x); // will print 96, the x outside of the method remains the same

Java Methods Inside of JSP Pages

We’re going to spend the next few labs creating methods inside of JSP pages. This is actually not a great practice, so we will move out of doing this shortly.

But for now, you need to know that using Java methods inside of a JSP requires some specific tags. Each created method needs to be wrapped inside its own set of <%! %> tags.

1. Make a file in ROOT\test called method.jsp.

2. Use the following code:

<!doctype html>
<html>
<head></head>
<body>
<%!
int addTen(int x) {
    x = x + 10;
    return (x);
}
%>
<%   
int x = 96; // even if the variable is the same name
out.println(addTen(x)); // will print 106
out.println(x); // will print 96, even though a 2nd variable named x becomes 106 inside of addTen
%>
</body>
</html>

3. Let’s walkthrough what’s happening when we call addTen(), in particular, the exact order of lines being executed by the program. We’ll use some line numbers below to making referencing easier:

01 int addTen(int x) {
02    x = x + 10;
03    return (x);
04 }
05
06 int x = 96; // even if the variable is the same name
07 out.println(addTen(x)); // will print 106
08 out.println(x); // will print 96

The program will run in this order:
Line 6
Line 7 - Starts
Line 1
Line 2
Line 3
Line 7 - Ends Line 8
